# 智学伴接口与并行开发契约

## 1. 文档用途

本文档是智学伴五人并行开发时共同遵守的接口与数据契约。各成员可以在依赖模块尚未完成时，按照本文档使用 Mock 数据开发；真实模块完成后，只替换数据来源，不再临时修改字段名称和业务含义。

本文档冻结以下内容：

- HTTP 接口路径、请求方式和认证要求；
- 请求与响应字段；
- 公共枚举和时间格式；
- 模块间 Service 调用和业务事件；
- 数据表归属和迁移编号；
- 幂等、分页、错误码和联调规则。

如果实现与本文档冲突，以本文档为准。确需调整时，必须由接口提供者、使用者和组长共同确认后修改本文档。

## 2. 模块负责人

| 成员 | 业务域 | Router 前缀 | Service 目录 | 前端接口文件 |
| --- | --- | --- | --- | --- |
| 成员一 | 用户画像、首次诊断、学习计划、短期记忆 | `/profile`、`/sessions` | `profile_service`、`diagnostic_service` | `api/profile.js` |
| 成员二 | 智能诊断、练习生成、内容质量校验 | `/learning` | `learning_service` | `api/learning.js` |
| 成员三 | 答题、掌握度、错题、订正和复习 | `/mastery`、`/mistakes` | `mastery_service`、`mistake_service` | `api/mastery.js` |
| 成员四 | 学习记录、推荐、每日计划和报告 | `/records`、`/recommendations`、`/reports` | `record_service`、`report_service` | `api/records.js`、`api/reports.js` |
| 成员五 | 积分、次数、VIP 和支付宝沙箱 | `/points`、`/vip` | `point_service`、`vip_service`、`payment_service` | `api/points.js`、`api/vip.js` |

每人负责本业务域的 Router、Schema、Service、DAO/Model、迁移脚本、测试和对应 Vue 页面。组长负责在 `backend/main.py` 注册 Router、处理公共文件冲突和最终联调。

## 3. 全局 HTTP 约定

### 3.1 基础地址

开发环境：

```text
http://127.0.0.1:8000
```

当前项目暂不统一添加 `/api/v1` 前缀，本轮新增接口直接使用本文档路径。后续如需版本前缀，由组长统一修改，成员不得自行添加不同前缀。

### 3.2 认证

除注册、登录、支付宝异步通知和健康检查外，所有接口必须携带：

```http
Authorization: Bearer <access_token>
```

业务中的当前用户只能从 JWT 解析。请求体或查询参数不得接收 `user_id` 作为权限依据。

### 3.3 成功响应

新增业务接口统一使用：

```json
{
  "code": "OK",
  "message": "success",
  "data": {}
}
```

无返回数据时：

```json
{
  "code": "OK",
  "message": "操作成功",
  "data": null
}
```

### 3.4 失败响应

```json
{
  "code": "PROFILE_NOT_COMPLETED",
  "message": "请先完善学习信息",
  "data": null
}
```

HTTP 状态码约定：

| 状态码 | 使用场景 |
| ---: | --- |
| 400 | 业务规则不满足、状态不允许 |
| 401 | 未登录、Token 无效或过期 |
| 403 | 已登录但无当前功能权益 |
| 404 | 当前用户范围内的数据不存在 |
| 409 | 重复提交或资源状态冲突 |
| 422 | 请求字段校验失败 |
| 500 | 未预期的服务器错误 |

### 3.5 分页

分页查询统一使用：

```text
page=1&page_size=20
```

限制：`page >= 1`，`1 <= page_size <= 100`。

分页响应 `data`：

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 0,
  "pages": 0
}
```

### 3.6 时间与日期

- API 时间统一返回带时区的 ISO 8601 字符串，例如 `2026-07-20T16:30:00+08:00`；
- 纯日期使用 `YYYY-MM-DD`；
- 数据库存储口径由组长统一，前端统一按照 `Asia/Shanghai` 展示；
- 每日次数和每日任务按 `Asia/Shanghai` 的自然日计算。

### 3.7 幂等

会产生积分、掌握度、次数、订单或 VIP 变化的写接口必须接受：

```http
X-Request-ID: <uuid>
```

后端以当前用户、业务类型和 `X-Request-ID` 建立唯一约束。相同请求重复到达时返回首次处理结果，不得重复修改数据。

## 5. 公共数据结构

### 5.1 学生画像摘要 `StudentProfileSummary`

```json
{
  "stage": "junior",
  "grade": "七年级",
  "subject": "数学",
  "learning_goal": "weakness",
  "weekly_study_days": 5,
  "daily_target_groups": 3,
  "diagnostic_status": "completed"
}
```

`diagnostic_status` 只能为 `required`、`in_progress`、`completed`、`skipped`。

### 5.2 知识点掌握摘要 `MasterySummary`

```json
{
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项",
  "mastery_score": 55,
  "learning_status": "weak",
  "answer_count": 12,
  "correct_count": 5,
  "last_studied_at": "2026-07-20T14:45:00+08:00"
}
```

### 5.3 题目 `Question`

```json
{
  "question_id": 1001,
  "content": "解方程：3x - 5 = 10",
  "question_type": "short_answer",
  "difficulty": "easy",
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项"
}
```

练习获取接口不得把 `standard_answer` 直接返回给前端。标准答案和解析只在提交答案后返回。

### 5.4 答题结果事件 `AnswerResultEvent`

这是成员二、三、四、五之间最重要的内部契约：

```json
{
  "request_id": "9a03580e-9c27-4c86-a21e-40eeb90aa721",
  "user_id": 1,
  "practice_id": 201,
  "question_id": 1001,
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项",
  "difficulty": "easy",
  "is_correct": false,
  "error_type": "calculation",
  "answered_at": "2026-07-20T14:45:00+08:00"
}
```

### 5.5 练习完成事件 `PracticeCompletedEvent`

```json
{
  "request_id": "0b0d3264-7771-463d-a5c8-b08bebc607de",
  "user_id": 1,
  "practice_id": 201,
  "subject": "数学",
  "knowledge_point_id": 101,
  "question_count": 3,
  "correct_count": 2,
  "accuracy": 66.67,
  "is_valid": true,
  "completed_at": "2026-07-20T14:48:00+08:00"
}
```

只有 `is_valid=true` 才能触发有效练习积分和每日目标进度。

### 5.6 错题订正完成事件 `CorrectionCompletedEvent`

```json
{
  "request_id": "219cc120-3335-4e08-ac71-5a8595eb7b0e",
  "user_id": 1,
  "mistake_id": 301,
  "knowledge_point_id": 101,
  "first_success": true,
  "completed_at": "2026-07-20T15:10:00+08:00"
}
```

只有 `first_success=true` 才能发放一次订正积分。

## 7. 成员一接口：画像、首次诊断与会话

### 7.1 查询当前画像

```http
GET /profile/me
```

响应 `data`：

```json
{
  "profile": {
    "stage": "junior",
    "grade": "七年级",
    "subject": "数学",
    "learning_goal": "weakness",
    "weekly_study_days": 5,
    "daily_target_groups": 3,
    "diagnostic_status": "completed"
  },
  "created_at": "2026-07-20T09:00:00+08:00",
  "updated_at": "2026-07-20T10:00:00+08:00"
}
```

尚未完善时返回 `data.profile=null`，HTTP 仍为 200，不使用 404。

### 7.2 保存当前画像

```http
PUT /profile/me
```

请求：

```json
{
  "stage": "junior",
  "grade": "七年级",
  "subject": "数学",
  "learning_goal": "weakness",
  "weekly_study_days": 5,
  "daily_target_groups": 3
}
```

规则：每周学习天数为 1～7，每日目标组数为 1～5；请求中不允许出现 `user_id`。

### 7.3 查询首次诊断状态

```http
GET /profile/diagnostic/status
```

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "status": "required",
    "diagnostic_id": null
  }
}
```

### 7.4 开始首次诊断

```http
POST /profile/diagnostic/start
```

响应 `data`：

```json
{
  "diagnostic_id": 501,
  "question_count": 5,
  "questions": [
    {
      "question_id": 5001,
      "content": "解方程：x + 3 = 7",
      "question_type": "short_answer",
      "difficulty": "easy",
      "knowledge_point_id": 101,
      "knowledge_point_name": "一元一次方程"
    }
  ]
}
```

### 7.5 提交或跳过首次诊断

```http
POST /profile/diagnostic/submit
POST /profile/diagnostic/skip
```

提交请求：

```json
{
  "diagnostic_id": 501,
  "answers": [
    {"question_id": 5001, "answer": "x=4"}
  ]
}
```

提交响应 `data`：

```json
{
  "status": "completed",
  "masteries": [
    {
      "knowledge_point_id": 101,
      "knowledge_point_name": "一元一次方程",
      "mastery_score": 60,
      "learning_status": "consolidating"
    }
  ]
}
```

跳过响应中 `status=skipped`，涉及的知识点初始掌握度统一为 60。

### 7.6 清除短期会话

```http
DELETE /sessions/{session_id}/memory
```

Redis 键统一为：

```text
user:{user_id}:session:{session_id}:short_term_memory
```

每个键最多保存 10 轮，每次写入刷新 24 小时 TTL。`user_id` 从 JWT 取得。

## 8. 成员二接口：诊断与练习生成

### 8.1 学情诊断

```http
POST /learning/diagnose
```

请求：

```json
{
  "session_id": 10001,
  "input_type": "weakness",
  "content": "我做一元一次方程时经常在移项处出错"
}
```

`input_type` 为 `question`、`learning_question`、`weakness` 之一。

响应 `data`：

```json
{
  "diagnosis_id": 601,
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项",
  "mastery_score": 55,
  "learning_status": "weak",
  "weakness": "移项后符号处理错误",
  "practice_suggestion": "先完成一组简单移项练习"
}
```

画像未完善时返回 400 和 `PROFILE_NOT_COMPLETED`。

### 8.2 创建练习组

```http
POST /learning/practices
```

请求头必须包含 `X-Request-ID`。

请求：

```json
{
  "diagnosis_id": 601,
  "question_count": 3
}
```

响应 `data`：

```json
{
  "practice_id": 201,
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项",
  "difficulty": "easy",
  "status": "in_progress",
  "questions": []
}
```

`questions` 使用公共 `Question` 结构。`question_count` 只能为 3～5。

生成成功后才由成员五扣减一次练习生成额度；生成失败不扣次数。调用成员五 Service 前先做权益预检，生成成功后再确认占用。

### 8.3 查询练习组

```http
GET /learning/practices/{practice_id}
```

只允许查询当前学生自己的练习。

### 8.4 提交练习答案

```http
POST /learning/practices/{practice_id}/answers
```

请求头必须包含 `X-Request-ID`。

请求：

```json
{
  "answers": [
    {"question_id": 1001, "answer": "x=5"},
    {"question_id": 1002, "answer": "x=-2"}
  ]
}
```

响应 `data`：

```json
{
  "practice_id": 201,
  "status": "completed",
  "question_count": 3,
  "correct_count": 2,
  "accuracy": 66.67,
  "results": [
    {
      "question_id": 1001,
      "is_correct": false,
      "standard_answer": "x=6",
      "analysis": "移项后常数项需要变号",
      "error_type": "calculation",
      "error_description": "移项时没有改变符号",
      "next_suggestion": "继续完成两道同类题"
    }
  ]
}
```

成员二完成判题后，为每道题构造 `AnswerResultEvent` 交给成员三；整组完成后构造 `PracticeCompletedEvent` 交给成员四和成员五。

## 9. 成员三接口：掌握度、错题和复习

### 9.1 查询知识点掌握情况

```http
GET /mastery/knowledge-points?page=1&page_size=20&status=weak
```

`status` 可省略，响应条目使用 `MasterySummary`。

### 9.2 查询掌握度趋势

```http
GET /mastery/trend?days=7
```

响应 `data`：

```json
{
  "current_score": 72,
  "change": 5,
  "points": [
    {"date": "2026-07-14", "score": 67},
    {"date": "2026-07-20", "score": 72}
  ]
}
```

### 9.3 查询错题

```http
GET /mistakes?page=1&page_size=20&status=pending
```

`status` 为 `pending`、`corrected`、`review_due`。

条目：

```json
{
  "mistake_id": 301,
  "question_id": 1001,
  "question_content": "解方程：3x-5=13",
  "user_answer": "x=5",
  "standard_answer": "x=6",
  "knowledge_point_id": 101,
  "knowledge_point_name": "一元一次方程-移项",
  "error_type": "calculation",
  "correction_status": "pending",
  "next_review_at": null,
  "created_at": "2026-07-20T14:45:00+08:00"
}
```

### 9.4 提交错题订正

```http
POST /mistakes/{mistake_id}/correction
```

请求头必须包含 `X-Request-ID`。

```json
{
  "answer": "x=6"
}
```

响应 `data`：

```json
{
  "mistake_id": 301,
  "is_correct": true,
  "correction_status": "corrected",
  "first_success": true,
  "review_dates": ["2026-07-21", "2026-07-23", "2026-07-27"]
}
```

首次订正成功后向成员五传递 `CorrectionCompletedEvent`。

### 9.5 查询今日复习

```http
GET /mistakes/reviews/today
```

返回当前日期到期且尚未完成的错题复习列表。

```json
  "question_count": 43,

  "created_at": "2026-07-20T16:30:00+08:00"
}
```

### 10.6 查询报告

```http
GET /reports?page=1&page_size=20
GET /reports/{report_id}
```

普通用户只能看到最近允许范围内的数据，VIP 可以查看全部；具体范围由成员五权益结果决定。

```http
GET /vip/status
GET /vip/usage
```

```json
  "expires_at": null

```

## 12. 后端内部 Service 契约

后端模块之间不通过 HTTP 调用本机接口，应直接调用 Service。为避免循环依赖，建议由学习流程编排层统一调用各 Service。

### 12.1 画像查询

```python
profile = await profile_service.get_summary(user_id)
```

返回 `StudentProfileSummary`，画像不存在时抛出：

```python
BusinessError("PROFILE_NOT_COMPLETED", "请先完善学习信息", 400)
```

### 12.2 答题结果处理

```python
mastery_result = await mastery_service.process_answer(answer_event)
```

返回：

```json
{
  "mastery_before": 58,
  "mastery_after": 55,
  "learning_status": "weak",
  "mistake_id": 301
}
```

### 12.3 完成练习后的编排顺序

```python
await record_service.record_practice(practice_event)
await plan_service.update_progress(practice_event)
await point_service.reward_practice(practice_event)
```

三个 Service 都必须用 `request_id` 实现幂等。其中单个非核心奖励失败，不应撤销已经保存的答题记录，但必须记录日志以便补偿。

### 12.4 权益预检和确认使用

生成练习之前：

```python
entitlement = await vip_service.check_entitlement(
    user_id=user_id,
    feature="practice_generation",
)
```

生成成功之后：

```python
await vip_service.consume_usage(
    user_id=user_id,
    feature="practice_generation",
    request_id=request_id,
)
```

生成失败时不得调用 `consume_usage`。

### 12.5 报告权益

```python
await vip_service.authorize_feature(
    user_id=user_id,
    feature="stage_report",
    payment_method="points",
    request_id=request_id,
)
```

VIP 直接通过；普通用户选择积分时由积分 Service 扣 20 分；积分不足抛出 `INSUFFICIENT_POINTS`。

## 13. 数据表归属与迁移编号

| 迁移 | 负责人 | 数据表 |
| --- | --- | --- |
| `002_profile_and_diagnostic.sql` | 成员一 | `user_profile` 扩展、`diagnostic_session`、`diagnostic_answer` |
| `003_learning_and_practice.sql` | 成员二 | `learning_session`、`diagnosis`、`practice`、`question` |
| `004_mastery_and_mistake.sql` | 成员三 | `answer_record`、`knowledge_mastery`、`mistake`、`review_plan` |
| `005_records_and_reports.sql` | 成员四 | `learning_record`、`daily_plan`、`notification`、`learning_report` |
| `006_points_vip_payment.sql` | 成员五 | `point_account`、`point_transaction`、`usage_record`、`vip_info`、`payment_order` |

规则：

- 不能多人修改同一个迁移文件；
- 外键列统一使用 `<对象名>_id`；
- 时间字段统一为 `created_at`、`updated_at`；
- 幂等业务表必须保存 `request_id` 并建立唯一索引；
- 金额字段使用 `DECIMAL(10,2)`；
- 不能使用迁移脚本删除已有业务数据；
- 跨成员外键由组长合并时统一检查。

## 14. 统一错误码

| 错误码 | HTTP | 负责人/场景 |
| --- | ---: | --- |
| `PROFILE_NOT_COMPLETED` | 400 | 画像未完善 |
| `DIAGNOSTIC_ALREADY_COMPLETED` | 409 | 重复提交首次诊断 |
| `INVALID_LEARNING_CONTENT` | 422 | 学习输入不符合要求 |
| `KNOWLEDGE_POINT_NOT_FOUND` | 400 | 无法识别知识点 |
| `PRACTICE_GENERATION_FAILED` | 500 | 练习生成或校验失败 |
| `PRACTICE_NOT_FOUND` | 404 | 练习不存在或不属于当前用户 |
| `PRACTICE_ALREADY_SUBMITTED` | 409 | 重复提交练习 |
| `MISTAKE_NOT_FOUND` | 404 | 错题不存在 |
| `CORRECTION_ALREADY_REWARDED` | 409 | 订正奖励已发放 |
| `INSUFFICIENT_POINTS` | 400 | 积分不足 |
| `USAGE_LIMIT_REACHED` | 403 | 达到每日使用上限 |
| `VIP_REQUIRED` | 403 | 功能要求 VIP |
| `ORDER_NOT_FOUND` | 404 | 订单不存在或不属于当前用户 |
| `ORDER_STATUS_CONFLICT` | 409 | 订单状态不允许当前操作 |
| `PAYMENT_VERIFY_FAILED` | 400 | 支付签名或金额校验失败 |
| `MODEL_SERVICE_UNAVAILABLE` | 503 | 模型服务暂不可用 |

错误码一旦被前端使用，不得随意改名。新增错误码应补充到本表。

## 15. Mock 数据与并行开发方式

### 15.1 成员二 Mock 画像

### 15.2 成员三 Mock 答题事件

### 15.3 成员四 Mock 完成事件

```text
PROFILE_COMPLETED
DAILY_LOGIN
PRACTICE_COMPLETED

```

事件载荷必须携带 `request_id`、`user_id` 和 `occurred_at`。

- 不允许为了联调同时兼容两套字段名；
- 上游完成后只替换 Fixture 或依赖注入实现；
- Mock 数据不得混入正式数据库迁移和生产默认数据。
